# Contributing to MeaningWorks

Thanks for your interest. This project is early stage — one developer, clear direction, moving fast.

---

## How to contribute

### Found a bug?
Open an issue. Include what you expected, what happened, and steps to reproduce.

### Have an idea?
Open an issue first. Describe the problem it solves and how it connects to existing functionality.

### Writing code?
1. Fork and branch from `main`
2. Write tests for new functionality
3. Follow existing patterns in the codebase
4. Submit a PR with a clear description

---

## Principles

These aren't style preferences. They're structural constraints:

- **Excavation, not invention** — every output must trace back to input
- **Provenance is non-negotiable** — every decision needs a trace
- **Bounded execution** — everything converges in finite steps at known cost
- **Determinism** — same input, same seed, same output
- **Unknowns are explicit** — hidden assumptions are bugs

- Integration with external tools and platforms
- Making the system accessible without dumbing it down
